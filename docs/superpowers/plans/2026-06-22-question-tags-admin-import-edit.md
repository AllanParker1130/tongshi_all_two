# 题库多标签导入与编辑实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 管理员端和教师端题库标签都支持多标签导入、编辑时可手动输入并修改标签；管理员公共课程题库导入不再要求 Excel 填写课程名称。

**Architecture:** 保持现有 `Question.tags` JSON 数组结构不变。教师端已有多标签输入和后端导入拆分能力，本次只做回归覆盖；管理员端补齐前端表单字段、API payload 类型、后端公共题目新增/编辑标签归一化和导入表头语义。

**Tech Stack:** Vue 3 `<script setup lang="ts">`、Element Plus、FastAPI、SQLAlchemy、pytest、Vite。

---

## 范围

- 修改管理员端公共课程题目新增/编辑弹窗，增加“课程标签”多选可创建输入。
- 修改管理员端公共课程题目新增/编辑 payload，发送 `tags: string[]`。
- 修改管理员公共课程题目导入界面说明，模板预览包含“标签”，不再展示“课程名称”。
- 修改管理员公共课程题目导入后端表头校验，不再要求“课程名称”。
- 修改管理员公共课程题目新增、编辑和导入服务，统一对标签去空格、拆分、去重，并同步到教师副本。
- 添加后端回归测试，覆盖管理员新增/编辑标签同步、管理员导入无课程名称且多标签拆分、教师导入多标签拆分。

## 不做

- 不变更数据库结构。
- 不新增标签字典、标签推荐或跨页自动补全。
- 不改变教师端公共同步题目只读规则。
- 不调整题库分页、搜索口径和导入失败详情结构。

## 验收

- 管理员端新增/编辑公共题目时，可以手动输入多个标签并保存。
- 管理员端公共课程导入 Excel 只需题型、标签、题干、选项、答案、解析；不需要课程名称列。
- 导入标签支持逗号、顿号、竖线、斜杠、分号分隔，保存为去重后的 `string[]`。
- 管理员修改公共题目标签后，已添加该公共课程的教师副本同步更新。
- 教师端现有新增/编辑/导入标签行为不回退。
